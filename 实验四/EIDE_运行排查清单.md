# EIDE 运行排查清单

适用现象：

- 工程已经在 EIDE 中成功编译
- `build/Debug/NewProject1.hex` 已生成
- 上电后 LCD 有显示
- 时间可能正常走动
- 温度显示异常，例如一直固定在 `76.7C`
- 超过上下限时蜂鸣器或 LED 没反应

---

## 1. 先确认程序是否真的在运行

观察 LCD 第一页：

```text
TIME 12:00:01
TEM 76.7C RUN
```

判断方法：

- 如果时间每秒都在变化，说明 `main.c` 主循环和 `Timer0` 中断基本正常
- 如果时间完全不动，优先排查烧录、晶振、芯片型号、定时器中断

当前项目里，只要时间在走，就说明不是“只编译了 i2c.c”，而是整个工程已经在运行。

---

## 2. 确认 EIDE 编译的是整个工程

只要点击的是 EIDE 的 `Build`，编译的就是整个工程，而不是当前打开的标签页。

编译日志中应能看到类似内容：

```text
CC '../alarm.c'
CC '../at24c08.c'
CC '../ds18b20.c'
CC '../main.c'
```

这说明 `main.c`、`ds18b20.c`、`i2c.c` 等都参与了编译。

---

## 3. 温度固定在 76.7C 时的判断

如果时间在走，但温度长期固定在 `76.7C`，通常不是正常室温，优先怀疑：

- `DS18B20` 接线错误
- `DQ` 没有上拉电阻
- 传感器没有真正接在代码设定的引脚
- 单总线时序不稳定，读到了固定错误值

当前代码中，`DS18B20` 配置为：

- `DQ -> P3.7`

定义位置：

- [config.h](/d:/Mono_chip_proj/实验四/config.h:45)

```c
sbit DS18B20_DQ = P3^7;
```

硬件检查：

- `DS18B20 VCC` 是否接 `5V`
- `DS18B20 GND` 是否接地
- `DS18B20 DQ` 是否接 `P3.7`
- `DQ` 是否通过 `4.7k` 电阻上拉到 `VCC`
- 传感器引脚顺序是否接反

---

## 4. 温度超过上限但不报警时的判断

当前显示：

- 当前温度：`76.7C`
- 上限：`40.5C`
- 下限：`20.0C`

按逻辑应进入高温报警。

报警代码判断规则：

- `cur_x10 > high_x10` 时，进入 `ALARM_HIGH`

相关代码位置：

- [alarm.c](/d:/Mono_chip_proj/实验四/alarm.c:46)

因此如果不报警，需要继续分成两类看：

### 4.1 先看 LCD 第 2 页

第二页应显示：

```text
ALARM:HIGH
```

如果 LCD 第 2 页显示 `ALARM:HIGH`：

- 说明报警逻辑已经成立
- 问题在蜂鸣器或 LED 硬件驱动

如果 LCD 第 2 页显示 `ALARM:NORMAL`：

- 说明报警状态没有被正确切换
- 需要重点检查 `g_temp_x10` 是否被正确读入，或报警状态更新流程

---

## 5. 蜂鸣器不响时重点检查

当前代码配置：

- 蜂鸣器引脚：`P1.5`
- 默认认为：`低电平有效`

定义位置：

- [config.h](/d:/Mono_chip_proj/实验四/config.h:55)

```c
sbit BUZZER = P1^5;
#define BUZZER_ACTIVE_LOW  1
```

如果你的开发板蜂鸣器实际是“高电平响”，那么当前配置就不会正常叫。

检查方法：

- 看开发板原理图，确认蜂鸣器是高电平触发还是低电平触发
- 如果是高电平触发，应把

```c
#define BUZZER_ACTIVE_LOW  1
```

改成

```c
#define BUZZER_ACTIVE_LOW  0
```

然后重新编译烧录。

---

## 6. LED 不闪时重点检查

当前代码配置：

- 高温报警 LED：`P1.0`
- 低温报警 LED：`P1.1`
- 默认认为：`低电平点亮`

定义位置：

- [config.h](/d:/Mono_chip_proj/实验四/config.h:59)

```c
sbit LED_HIGH = P1^0;
sbit LED_LOW  = P1^1;
#define LED_ACTIVE_LOW     1
```

如果你的开发板 LED 实际是高电平点亮，则也需要改配置。

---

## 7. 推荐排查顺序

1. 先确认时间是否在走
2. 切到 LCD 第 2 页，看是否显示 `ALARM:HIGH`
3. 查看高温报警 LED 是否闪烁
4. 确认蜂鸣器接法和高低电平有效方式
5. 确认 `DS18B20` 是否真的接在 `P3.7`
6. 确认 `DQ` 有 `4.7k` 上拉电阻

---

## 8. 当前现象的高概率结论

结合目前现象：

- 时间在走
- 温度一直固定 `76.7C`
- 上限 `40.5C`
- 没有叫声

高概率是两个问题同时存在：

1. `DS18B20` 通信不正常，导致温度读数是假值
2. 蜂鸣器有效电平与代码配置不一致，或者蜂鸣器不在 `P1.5`

---

## 9. 建议下一步

建议先做两个最小验证：

1. 看 LCD 第 2 页是否显示 `ALARM:HIGH`
2. 看开发板原理图，确认：
   - `DS18B20` 是否接 `P3.7`
   - 蜂鸣器是否接 `P1.5`
   - 蜂鸣器是否低电平有效

如果需要进一步定位，建议加两个调试功能：

- `DS18B20` 读不到时，LCD 显示 `ERR`
- 上电时让蜂鸣器自检短响一次

这样可以快速分清是传感器问题还是蜂鸣器驱动问题。
