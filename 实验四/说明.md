# 实验五：基于 DS18B20 的温度传感器设计 — 工程说明

> 平台：普中 HC6800-ES V2.0 (STC89C52RC) + Keil μVision4/5 + Proteus 8
> 晶振：11.0592 MHz   串口：9600 bps   8 路独立时基均由 Timer0 产生

---

## 一、文件结构

```
实验四/
├── config.h        # 引脚 / 类型 / 全局变量 / 常量集中定义
├── main.c          # 系统初始化 + 主循环调度 + LCD 页面渲染
├── timer.c/.h      # Timer0 1ms 节拍 (1s/50ms/100ms/300ms)
├── lcd1602.c/.h    # LCD1602 8 位并行驱动 (D0-D7=P0)
├── ds18b20.c/.h    # DS18B20 单总线驱动 (非阻塞: StartConvert + Read)
├── i2c.c/.h        # 软件 I2C (SCL=P2^1, SDA=P2^0)
├── at24c08.c/.h    # AT24C08 EEPROM, 参数区 + 循环记录区
├── uart.c/.h       # 串口中断收发 + 缓冲区管理
├── command.c/.h    # 串口命令解析 (start/stop/T=/TEM_*/TIME=)
├── key.c/.h        # KEY1/KEY2 状态机 (短按 +0.5 / 长按 -0.5)
├── alarm.c/.h      # 蜂鸣器 + LED 双频报警 (高温 5Hz / 低温 1.67Hz)
└── 实验要求.txt
```

## 二、Keil 工程建立步骤

1. 新建 Keil C51 项目，选 `STC89C52RC` (或 AT89C52，无 SFR 区别)
2. 把上面 11 对 `.c/.h` 全部加入 `Source Group 1`
3. Target Options → Output → 勾选 **Create HEX File**（生成 `.hex` 用于烧录/仿真）
4. C51 → Optimization 默认 8，可保持；勾选 Browse Information 便于跳转
5. 直接 `Build`，无浮点、无 stdio，C51 v9.x 编译应 0 警告

## 三、Proteus 接线表

| 外设 | MCU 引脚 | 备注 |
|---|---|---|
| LCD1602 D0..D7 | P0.0..P0.7 | **P0 必须加 10k 排阻上拉到 VCC** |
| LCD1602 RS / RW / EN | P2.6 / P2.5 / P2.7 | V0 接 10k 电位器调对比度 |
| DS18B20 DQ | P3.7 | **DQ 加 4.7k 上拉到 VCC** |
| AT24C08 SCL / SDA | P2.1 / P2.0 | **SCL/SDA 各加 4.7k 上拉**；A0/A1/A2/WP 接 GND |
| KEY1 / KEY2 | P3.2 / P3.3 | 一端接 MCU 引脚，另一端接 GND |
| BUZZER | P1.5 | 通过 8550/9012 三极管驱动有源蜂鸣器（低有效） |
| LED_HIGH / LED_LOW | P1.0 / P1.1 | 串 470Ω 限流电阻接 VCC（低电平点亮） |
| RXD / TXD | P3.0 / P3.1 | 接 COMPIM (Proteus 虚拟串口) 或 MAX232 |

> 普中开发板上述外设已硬连，无需改动；P0 的上拉、I2C 的上拉电阻板上已自带。

## 四、AT24C08 地址布局（详细）

```
0x000             : 魔术字 0x5A           (1B)
0x001 ~ 0x002     : g_temp_high_x10       (s16, low first)
0x003 ~ 0x004     : g_temp_low_x10        (s16, low first)
0x005             : g_upload_interval     (1B, 1..8)
0x006             : g_ee_write_idx        (1B, 0..200)
0x007 ~ 0x00F     : 保留
─────────────────────────────────────────────────────
0x010 起          : 温度记录区, 5 字节/条, 共 201 条
  [hour] [min] [sec] [temp_x10_low] [temp_x10_high]
─────────────────────────────────────────────────────
共占用: 0x010 + 201*5 = 0x3F5, 末尾 0x3F5..0x3FF 共 11B 不用
循环规则: 第 201 次写完回到 idx=0 覆盖
```

## 五、串口命令一览

| 命令 | 示例 | 应答 |
|---|---|---|
| `start_tem` | start_tem\r\n | `OK:START` |
| `stop_tem` | stop_tem\r\n | `OK:STOP` |
| `T=X` (X∈[2,8]) | T=3\r\n | `OK:T=3` |
| `TEM_L=XX.X` | TEM_L=18.5\r\n | `OK:TEM_L=18.5C` |
| `TEM_H=XX.X` | TEM_H=32.0\r\n | `OK:TEM_H=32.0C` |
| `TIME=hh:mm:ss` | TIME=12:30:00\r\n | `OK:TIME=12:30:00` |

非法 → `ERROR:VALUE` (参数越界/格式错)，未识别 → `ERROR:CMD`

> 默认上传周期 1s。`T=X` 仅接受 2..8（实验要求 1<X<9）。
> 想恢复 1s 上传，重启或 stop_tem→start_tem 后默认值 1 仍生效。

每秒上传一帧示例：
```
TIME=12:30:05,TEM=25.6C,TEM_L=20.0C,TEM_H=30.0C,STATE=RUN
```

## 六、按键交互

| 操作 | 行为 |
|---|---|
| KEY1 短按 | 切换调节对象：NONE → LOW → HIGH → NONE，并自动跳到限值页 |
| KEY2 短按（<800ms） | 当前选中阈值 **+0.5℃** |
| KEY2 长按（≥800ms） | 当前阈值 **-0.5℃**，每 200ms 自动重复 |

调节后立刻：① LCD 同步刷新；② 写入 AT24C08 参数区。

## 八、测试步骤

### 1. 温度显示
- 上电后 LCD 第二行显示 `TEM 25.6C RUN`（值取决于 DS18B20）
- 用手指接触 DS18B20 → 温度上升、LCD 同步更新、串口数据变化

### 2. 负温度
- Proteus 中右键 DS18B20 → `Temperature` 设为 `-3.5`
- LCD 应显示 `TEM -3.5C`，串口 `TEM=-3.5C`

### 3. 高温报警
- `TEM_H=20.0` 设低上限 → 当前温度大于 20.0
- LED_HIGH (P1.0) 5Hz 闪烁，蜂鸣器 5Hz 急促响
- LCD 切到限值页应显示 `ALARM:HIGH`

### 4. 低温报警
- `TEM_L=50.0` 设高下限 → 当前温度小于 50.0
- LED_LOW (P1.1) ~1.67Hz 慢闪，蜂鸣器频率明显比高温慢
- LCD 显示 `ALARM:LOW`

### 5. 按键调节
- 按 KEY1 一次 → LCD 跳到限值页，KEY SET:LOW
- 按 KEY2 一次 → 下限 +0.5℃ 同步显示
- 按住 KEY2 ≥ 1 秒 → 下限连续 -0.5℃
- 再按 KEY1 → 切到 HIGH，调上限；第三次回到 NONE

### 6. 串口控制
PC 端打开串口工具（9600/N/8/1），逐条发送：
```
start_tem<CR>     ← OK:START，恢复采集和上传
stop_tem<CR>      ← OK:STOP，串口停止上传，LCD 显示 STOP
T=3<CR>           ← OK:T=3，每 3 秒上传一次
TEM_L=18.5<CR>    ← OK:TEM_L=18.5C
TEM_H=32.0<CR>    ← OK:TEM_H=32.0C
TIME=12:30:00<CR> ← OK:TIME=12:30:00
TEM_H=999.9<CR>   ← ERROR:VALUE
hello<CR>         ← ERROR:CMD
```

### 7. EEPROM 验证
- Proteus 中右键 AT24C08 → `Memory Contents` 可查看实时字节
- 0x000 应显示 `5A`（魔术字）
- 0x010 起每秒变化 5 字节
- 关电后再上电，上次设置的上下限和上传周期保持不变

## 九、未实现项 / 偏离说明

1. **温度曲线（选做）**：未实现。LCD1602 是 5×8 字符液晶，没有逐像素绘图能力，
   不适合连续曲线。如需曲线请改用 SSD1306 OLED12864 + 自建像素缓冲。
2. **LCD 数据线模式**：实验要求建议 4 位模式 (D4-D7=P0^4..P0^7)，
   本工程沿用既有实验三的 8 位模式 (D0-D7=P0)，引脚定义集中于 `config.h`，
   如需切到 4 位模式，仅需重写 `lcd1602.c` 中 `LCD_WriteCmd/Data` 的字节拆分逻辑。
3. **EEPROM 1Hz 写入**：实际工程中 AT24C08 写入寿命约 1,000,000 次，按 1Hz
   持续写约 11 天即耗尽。本实验仅按要求实现，已在 `at24c08.h` 与 `main.c`
   注释中明确标注。

## 十、定时器/中断分配

| 资源 | 用途 |
|---|---|
| Timer0 | 1ms 系统节拍（1s / 50ms / 100ms / 300ms 派生） |
| Timer1 | UART 9600 bps 波特率（mode 2 自动重装 0xFD） |
| INT 1 (Timer0) | `Timer0_ISR` |
| INT 4 (UART) | `UART_ISR` |
| INT0/INT1 | 未启用，P3.2/P3.3 仅作 GPIO 用 |
